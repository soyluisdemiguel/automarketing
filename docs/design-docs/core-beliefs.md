# Core Beliefs

- Documentation is part of the product delivery system.
- Explicit constraints beat implied conventions.
- User value should appear before platform elaboration.
- Observable systems are easier to improve than opaque ones.
- Simple defaults are better than optional complexity.
- Plans, specs, and design docs should agree or clearly disagree on purpose.
